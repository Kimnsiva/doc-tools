โปรเจคนี้คือ **ระบบจัดการเอกสารบัญชี (Accounting Document Management System - DMS)** ที่ถูกออกแบบมาเพื่อช่วยสแกนและจัดเก็บเอกสารอย่างเป็นระบบ โดยมีจุดเด่นคือการใช้ AI เข้ามาช่วยอ่านและดึงข้อมูลจากเอกสารอัตโนมัติ

นี่คือรายละเอียดและส่วนประกอบสำคัญของโปรเจคนี้ครับ:

### 🌟 ภาพรวมและฟีเจอร์หลัก (Project Overview)
โปรเจคนี้ใช้สำหรับนำเข้าเอกสารทางการบัญชี (ไฟล์ PDF หรือรูปภาพ) ระบบจะทำการประมวลผลด้วย **EasyOCR** เพื่ออ่านข้อความ (รองรับทั้งภาษาไทยและอังกฤษ) และดึงข้อมูลฟิลด์ที่สำคัญ เช่น ชื่อ, เลขสัญญา, และที่อยู่ ออกมาโดยอัตโนมัติ 
จากนั้นผู้ใช้สามารถตรวจสอบข้อมูลและบันทึกลงในระบบ ซึ่งระบบจะจัดเก็บไฟล์อย่างเป็นระเบียบตามวันที่ (เช่น `DMS/2026/07/15/`) พร้อมบันทึกข้อมูล Metadata ลงฐานข้อมูลสำหรับการค้นหาย้อนหลังแบบ Full-text search

### 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)
*   **AI & OCR Engine:** `EasyOCR` สำหรับสกัดข้อความจากภาพ และ `PyMuPDF (fitz)` สำหรับดึงข้อความจาก Digital PDF โดยตรง (และใช้เรนเดอร์ PDF เป็นภาพความละเอียด 300 DPI)
*   **Backend:** `FastAPI` + `Uvicorn` เป็น REST API Server (รันที่พอร์ต 8000)
*   **Database:** `SQLite3` พร้อมใช้งานฟีเจอร์ `FTS5` เพื่อให้สามารถค้นหาข้อความในเอกสารได้อย่างรวดเร็วและแม่นยำ
*   **Frontend (มี 2 รูปแบบ):**
    1.  **Static SPA (Primary):** เป็นไฟล์ HTML, Vanilla JS, CSS ธรรมดา ที่ถูก Serve ออกมาผ่าน FastAPI โดยตรง
    2.  **Streamlit App:** เป็น UI ทางเลือกสำหรับผู้ใช้ที่รันผ่าน Streamlit (รันที่พอร์ต 5550)
*   **Deployment:** รองรับการรันผ่าน Virtual Environment (`.venv`) และปัจจุบันมีการกำหนดค่า `docker-compose.yml` ไว้สำหรับรันผ่าน Docker เต็มรูปแบบ (ซึ่งคุณกำลังรัน `docker-compose up --build` อยู่ในขณะนี้)

### 📂 โครงสร้างโฟลเดอร์ที่สำคัญ
*   `backend/`: ส่วนประกอบของ API และ Database
    *   `main.py`: โค้ดหลักของ API ที่ใช้จัดการการสกัดข้อความ, จัดการไฟล์, และเชื่อมต่อฐานข้อมูล
    *   `dms.db`: ฐานข้อมูล SQLite ที่ระบบสร้างขึ้นอัตโนมัติ
*   `frontend/`: โค้ดส่วนหน้าจอผู้ใช้งาน
    *   `index.html`, `app.js`, `style.css`: หน้าจอผู้ใช้งานแบบหลัก
    *   `streamlit_app.py`: หน้าจอผู้ใช้งานแบบ Streamlit
*   `DMS/`: โฟลเดอร์สำหรับจัดเก็บเอกสารจริงที่บันทึกเข้าระบบ (จะถูกสร้างและจัดกลุ่มโฟลเดอร์ย่อยตาม ปี/เดือน/วัน)
*   `storage/`: โฟลเดอร์สำหรับเก็บไฟล์ชั่วคราวระหว่างการประมวลผล
*   `docker-compose.yml`: ไฟล์กำหนดค่าสำหรับรันทั้งระบบขึ้นมาพร้อมกันด้วย Docker (แบ่งคอนเทนเนอร์เป็น backend และ frontend)

### 🔄 ขั้นตอนการทำงาน (Workflow)
1.  **Upload:** ผู้ใช้อัปโหลดเอกสารผ่านหน้าเว็บ
2.  **Pre-process & OCR:** ระบบวิเคราะห์เอกสาร ถ้าระบุตัวอักษรใน PDF ได้โดยตรงจะดึงออกมาเลย (ทำงานได้เร็ว) แต่ถ้าเป็นรูปภาพจะส่งเข้า EasyOCR เพื่ออ่านข้อความภาษาไทย/อังกฤษ
3.  **Extraction:** ระบบสกัดข้อมูลสำคัญตาม Keyword เช่น "วันที่", "ชื่อ" ออกมาเป็นฟิลด์ต่างๆ
4.  **Review:** ผู้ใช้ตรวจสอบความถูกต้องและแก้ไขข้อมูลบนหน้าจอได้ก่อนบันทึก
5.  **Save & Index:** ระบบนำไฟล์ไปเก็บในโฟลเดอร์ `DMS` และนำข้อมูลไปบันทึกลง SQLite
6.  **Search:** ผู้ใช้สามารถค้นหาเอกสารเก่าๆ ได้จาก Keyword หรือช่วงวันที่ผ่านระบบ Library บนหน้าเว็บ

หากคุณต้องการให้ผมปรับปรุงโค้ดส่วนไหน เพิ่มฟีเจอร์อะไร หรือติดปัญหาตรงไหน สามารถแจ้งได้เลยครับ!
